# Руководство разработчика — ЭЭГ-анализатор

## 1. Состав проекта

```
EEG/
├── target_fragment_analysis.py   # ЯДРО: расчёт (детекция, PSD, коэффициенты, графики)
├── worker.py                     # backend для C++ GUI (режимы load / analyze / serve)
├── eeg_gui.py                    # GUI на Python (PyQt6) — самостоятельная версия
├── cpp_gui/                      # GUI на C++/Qt6
│   ├── CMakeLists.txt
│   ├── main.cpp                  # точка входа, --selftest
│   ├── mainwindow.{h,cpp}        # окно, логика, запуск воркера, разбор JSON
│   ├── signalview.{h,cpp}        # виджет сигнала + выделение мышью (QPainter)
│   └── imageview.{h,cpp}         # показ PNG с сохранением пропорций
├── stable_freq.py, alpha_detect.py  # ранние прототипы детекции
├── NP042309_CE.fif               # пример записи
├── results/ figures/ .gui_cache/ # выходные данные
└── docs/                         # эта документация
```

Ключевая идея архитектуры: **вся «математика» живёт в одном месте**
(`target_fragment_analysis.py`), а интерфейсы её переиспользуют, не дублируя.

---

## 2. Поток данных (pipeline)

```
.fif → read_raw_fif → filter(0.5–30) → set_eeg_reference("average")
     → детекция устойчивых фрагментов (по O1,O2,P3,P4)
     → выбор фрагмента [tmin, tmax]
     → crop → по каждому каналу: Welch PSD → коэффициенты альфа-ритма
     → таблица + спектр + спектрограмма + топокарта
```

## 3. Ядро: `target_fragment_analysis.py`

### 3.1. Константы (вверху файла)

| Имя | Значение | Смысл |
|---|---|---|
| `DETECT_CH` | `["O1","O2","P3","P4"]` | каналы для детекции устойчивости |
| `BAND` | `(6.0, 15.0)` | полоса поиска доминирующей частоты |
| `MIN_DURATION` | `1.0` | мин. длительность устойчивого участка, с |
| `FREQ_TOL` | `0.5` | допуск отклонения частоты, ±Гц |
| `WIN_SEC` | `2.0` | окно измерения частоты, с |
| `STEP_SEC` | `0.25` | шаг окна, с |
| `ANALYSIS_BAND` | `(1.0, 30.0)` | полоса спектрального анализа |
| `ALPHA_BAND` | `(8.0, 13.0)` | альфа-полоса |
| `PSD_NFFT` | `2048` | длина FFT (густота частотной сетки) |
| `BANDS` | delta/theta/alpha/beta | границы ритмов |

### 3.2. Функции

- **`dominant_freq_track(sig, sfreq)`** — строит спектрограмму скользящим окном
  и возвращает «дорожку» доминирующей частоты во времени `(t, freq, power)`.
- **`find_stable_runs(t, dom_freq, gate, min_dur, tol)`** — жадно склеивает
  подряд идущие моменты, пока частота не уходит дальше `tol` от средней по
  накопленному участку; возвращает список устойчивых интервалов.
- **`detect_target_fragment(raw, sfreq)`** — усредняет каналы детекции,
  получает дорожку, ставит порог по мощности (`gate = power > median`),
  находит все интервалы и выбирает **самый длинный** (целевой).
- **`channel_psd(sig, sfreq)`** — спектр одного канала методом Уэлча
  (`nperseg=min(len, 2*sfreq)`, `nfft=max(2048, nperseg)`), обрезанный до 1–30 Гц.
- **`band_power(f, p, lo, hi)`** — мощность в полосе как интеграл (трапеции) PSD.
- **`alpha_coefficients(f, p)`** — все характеристики альфа-ритма по спектру
  (см. таблицу метрик в `worker.py` / `mainwindow.cpp`).
- **`plot_topomaps`, `plot_spectra`, `main`** — пакетный режим (CLI), пишет в
  `figures/` и `results/`.

> Запуск как самостоятельный скрипт:
> `python target_fragment_analysis.py` — посчитает целевой фрагмент и сохранит
> CSV/PNG. `main()` срабатывает только при `__main__`, поэтому импорт модуля
> безопасен (GUI этим пользуются).

### 3.3. Единицы

MNE отдаёт данные в **вольтах**. PSD получается в В²/Гц, поэтому при выводе
мощности в мкВ² умножаем на `1e12`. Амплитуда сигнала на графиках — `*1e6`
(В → мкВ).

## 4. Backend для C++: `worker.py`

CLI с тремя режимами:

```bash
python worker.py load    --file F.fif --out DIR
python worker.py analyze --file F.fif --tmin A --tmax B --channels c1,c2 --metric M --out DIR
python worker.py serve   # постоянный процесс, команды JSON-строками из stdin
```

- **`do_load(raw, out)`** — каналы, sfreq, длительность, прореженный сигнал для
  отрисовки, список устойчивых интервалов, целевой → `result.json`.
- **`do_analyze(raw, tmin, tmax, channels, metric, out)`** — коэффициенты по
  каналам, `coefficients.csv`, картинки `spectrum.png` / `spectrogram.png` /
  `topomap.png` → `result.json`.
- **`serve()`** — держит **одну** запись в кэше (`cache`), на каждую команду
  пишет `result.json` и печатает строку `DONE`. Это убирает повторное чтение
  `.fif` (главный источник лагов).

Формат обмена — **файлы**: воркер пишет `result.json` (и PNG), GUI его читает.
Через stdout идёт только сигнал готовности `DONE`, чтобы вывод matplotlib/MNE не
ломал парсинг.

Русские подписи метрик — словарь `METRIC_RU` (заголовок топокарты).

## 5. GUI на C++/Qt6: `cpp_gui/`

### 5.1. Сборка

CMake + AUTOMOC, `find_package(Qt6 COMPONENTS Widgets)`. Цель — `eeg_gui_cpp`.
Самопроверка без дисплея:

```bash
QT_QPA_PLATFORM=offscreen ./build/eeg_gui_cpp --selftest
```

(`--selftest` загружает файл по умолчанию, считает целевой фрагмент и выходит —
прогоняет всю цепочку C++ ↔ Python.)

### 5.2. `MainWindow` (mainwindow.cpp)

- **Конфигурация**: `python_`, `projDir_`, `worker_`, `cacheDir_` — берутся из
  переменных окружения `EEG_PYTHON` / `EEG_PROJECT_DIR` или из значений по
  умолчанию.
- **`startServer()`** — поднимает `QProcess` (`python -u worker.py serve`) и
  держит его живым весь сеанс.
- **`sendCommand(QJsonObject)`** — пишет JSON-команду в stdin, ждёт `DONE`,
  затем `readResult()` читает `result.json`. Запись `.fif` не перечитывается.
- **`loadFile()`** — команда `load`; заполняет список каналов, сигнал,
  устойчивые интервалы, таблицу фрагментов; сразу считает целевой.
- **`onCalc()`** — команда `analyze`; заполняет таблицу, сводку, картинки.
- **Список каналов**: `NoSelection` + `NoFocus`, клик по строке переключает
  галочку (`itemClicked`). Пункты имеют только флаг `ItemIsEnabled`, поэтому
  индикатор сам не переключается — нет двойного срабатывания.
- **Метрики**: `kMetrics` — соответствие английский ключ → русская подпись;
  `metricRu()` локализует заголовки; в `QComboBox` подпись русская, а в воркер
  уходит ключ (через `itemData`).
- **Ручное время**: `tminSpin_` / `tmaxSpin_` + `onApplyManualTime()`;
  `updateSelectionUi()` синхронизирует график, поля и подпись.
- **Таблица фрагментов**: `runsInfo_`, сортировка по длительности, целевой
  подсвечен; двойной клик → переход к фрагменту.

### 5.3. `SignalView` (signalview.cpp)

Собственный виджет на `QPainter`:
- рисует сигнал, зелёные полосы устойчивых интервалов, оранжевую — целевой;
- **кэширует фон** (сигнал + интервалы) в `QPixmap`; при перетаскивании
  перерисовывается только синяя рамка выделения — отсюда плавность;
- `mousePress/Move/Release` → выбор `[tmin, tmax]` → сигнал `selectionChanged`.

### 5.4. `ImageView` (imageview.cpp)

`QLabel`, хранит исходный `QPixmap` и масштабирует под размер с сохранением
пропорций (`resizeEvent`).

## 6. GUI на Python: `eeg_gui.py`

Самостоятельная версия на PyQt6 с той же логикой, но без воркера: расчёт
вызывается напрямую (импорт функций из `target_fragment_analysis.py`),
графики — через `matplotlib` `FigureCanvasQTAgg`. Удобна для быстрых проверок.

## 7. Как менять поведение

| Хочу | Где править |
|---|---|
| другую альфа-полосу (напр. 7–11 Гц) | `ALPHA_BAND` в `target_fragment_analysis.py` |
| другие каналы детекции | `DETECT_CH` |
| мягче/строже критерий устойчивости | `FREQ_TOL`, `MIN_DURATION`, порог `gate` |
| точнее частотную сетку | `PSD_NFFT` |
| новый коэффициент | добавить в `alpha_coefficients()` + `METRIC_KEYS`/`kMetrics` |
| русский CSV | переименовать колонки в `do_analyze()` |
| путь к python/проекту (C++) | env `EEG_PYTHON`, `EEG_PROJECT_DIR` |

После правки `kMetrics` (C++) и `METRIC_KEYS`/`METRIC_RU` (Python) держать
синхронными.

## 8. Известные ограничения

- Модальная частота ищется **внутри** `ALPHA_BAND`; если реальный пик вне полосы,
  он будет «прижат» к границе — следить за соответствием полосы данным.
- Порог `gate = power > median` пропускает ~половину моментов — эвристика, не
  абсолютный критерий значимости.
- `find_stable_runs` сравнивает с **бегущим средним** — возможен медленный дрейф
  частоты в пределах одного «забега».
- Топокарта требует ≥4 каналов с координатами (монтаж в `raw.info`).
- C++ GUI блокирует UI на время расчёта (синхронный `sendCommand`); для записей
  обычного размера это доли секунды.
