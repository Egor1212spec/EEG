# Руководство пользователя — ЭЭГ-анализатор фрагментов

## 1. Что делает программа

Программа помогает проанализировать запись ЭЭГ и количественно описать
**альфа-ритм** на её отдельных участках. Она умеет:

- открыть запись ЭЭГ в формате `.fif`;
- автоматически найти **устойчивые интервалы** — участки, где частота
  доминирующего ритма мало меняется во времени;
- показать сигнал и подсветить эти интервалы цветом;
- позволить выделить **любой** фрагмент (мышью или вводом времени) и
  посчитать по нему набор характеристик;
- построить **спектр**, **спектрограмму** и **карту распределения по голове**
  (топокарту);
- выгрузить коэффициенты в CSV-таблицу.

Есть две версии интерфейса с одинаковой логикой:

- **`eeg_gui.py`** — на Python (PyQt6), запускается одной командой;
- **`cpp_gui/`** — на C++/Qt6 (быстрее, расчёт выполняет Python-воркер в фоне).

## 2. Установка и запуск

### 2.1. Окружение

Все расчётные библиотеки (MNE, NumPy, SciPy, pandas, matplotlib) установлены в
conda-окружение **`eeg`**. Проверить:

```bash
conda activate eeg
python -c "import mne, scipy, pandas; print('OK')"
```

### 2.2. Python-версия (проще всего)

```bash
conda activate eeg
python eeg_gui.py
```

Файл `NP042309_CE.fif` подхватится автоматически, если лежит рядом.

### 2.3. C++-версия

Нужны g++, CMake и Qt6 (уже стоят на рабочей машине). Сборка:

```bash
cd cpp_gui
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release   # один раз
cmake --build build -j4
./build/eeg_gui_cpp
```

> ⚠️ Обязательно `-DCMAKE_BUILD_TYPE=Release` — без оптимизации интерфейс
> работает заметно медленнее.

Если проект лежит не в `/home/maths/vscode/EEG`, путь и интерпретатор
задаются переменными окружения:

```bash
EEG_PYTHON=/путь/к/envs/eeg/bin/python \
EEG_PROJECT_DIR=/путь/к/проекту \
./build/eeg_gui_cpp
```

## 3. Интерфейс

```
┌───────────────┬───────────────────────────────────────────┐
│  Левая панель │   Сигнал (выделение фрагмента мышью)       │
│  управления   │   зелёным — устойчивые интервалы           │
│               │   оранжевым — целевой (самый длинный)      │
│               ├───────────────────────────────────────────┤
│               │   Вкладки результатов:                     │
│               │   • Коэффициенты                           │
│               │   • Устойчивые фрагменты                   │
│               │   • Спектр (PSD)                           │
│               │   • Спектрограмма                          │
│               │   • Голова (карта)                         │
└───────────────┴───────────────────────────────────────────┘
```

### 3.1. Левая панель

| Элемент | Назначение |
|---|---|
| **Открыть .fif…** | загрузить запись ЭЭГ |
| **Список каналов** | галочками выбрать отведения для расчёта; клик по строке включает/выключает канал |
| **Все / Снять / Затылочные** | быстрый выбор каналов |
| **Метрика для карты головы** | какой коэффициент показать на топокарте |
| **Время фрагмента вручную** | поля «с … по …» и кнопка «Задать фрагмент по времени» |
| **Рассчитать по фрагменту** | посчитать по текущему выделению |
| **Взять целевой фрагмент** | подставить самый длинный устойчивый интервал |
| **Экспорт CSV** | сохранить таблицу коэффициентов |

### 3.2. Выделение фрагмента

Три способа задать фрагмент:

1. **Мышью** — потянуть по горизонтали на верхнем графике.
2. **Вручную** — ввести начало и конец в полях «с … по …» и нажать
   «Задать фрагмент по времени».
3. **Из списка** — на вкладке «Устойчивые фрагменты» дважды кликнуть строку.

Поля времени и рамка на графике синхронизированы: меняешь одно — обновляется
другое.

### 3.3. Вкладки результатов

- **Коэффициенты** — таблица по каждому выбранному отведению + сводка
  (средняя модальная частота ± разброс, альфа-индекс, канал с максимумом альфы).
- **Устойчивые фрагменты** — все найденные интервалы (№, начало, конец,
  длительность, частота), отсортированы по длительности; целевой подсвечен.
  Двойной клик — перейти к фрагменту и посчитать.
- **Спектр (PSD)** — спектры выбранных каналов с подсветкой альфа-полосы.
- **Спектрограмма** — как меняется частотный состав во времени внутри фрагмента.
- **Голова (карта)** — топокарта выбранной метрики (нужно ≥4 канала).

## 4. Что означают коэффициенты

| Коэффициент | Смысл |
|---|---|
| **Модальная частота** | частота с максимумом мощности в альфа-полосе |
| **Центроид** | «центр тяжести» спектра альфы (средневзвешенная частота) |
| **Частотный разброс** | насколько ритм «размазан» по частоте (СКО) |
| **Ширина пика (FWHM)** | ширина альфа-пика на половине высоты |
| **Альфа-индекс, %** | доля мощности альфы во всём спектре 1–30 Гц |
| **Мощность альфы** | абсолютная мощность в полосе 8–13 Гц, мкВ² |
| **Альфа/тета, Альфа/бета** | соотношения мощностей ритмов |
| **Пиковость** | насколько пик выделяется над средним фоном альфы |
| **Добротность** | острота пика = модальная частота / FWHM |
| **Спектральная энтропия** | мера «беспорядка»: 0 — чистый пик, 1 — шум |

Подробное объяснение метода — в `docs/DEVELOPER_GUIDE.md`.
Вопросы и ответы по устройству самого приложения (Qt) — в `docs/QA.md`.

## 5. Типовой сценарий работы

1. Открыть `.fif`.
2. Программа сама находит устойчивые интервалы и считает самый длинный.
3. Посмотреть список на вкладке «Устойчивые фрагменты».
4. При необходимости выбрать другой фрагмент (мышью / вручную / двойным кликом).
5. Отметить нужные каналы (по умолчанию — все 31).
6. Нажать «Рассчитать».
7. Изучить таблицу, спектр, спектрограмму, топокарту.
8. «Экспорт CSV» — сохранить результат для отчёта/ментора.

## 7. Частые вопросы

**Сколько длится фрагмент для анализа?**
Минимум ~1 с; оптимально 2–5 с — достаточно для устойчивой оценки спектра.

**Почему альфа-пик показывается на 8 Гц, хотя «на глаз» он ниже?**
По умолчанию альфа-полоса задана 8–13 Гц. Если у испытуемого ритм смещён
(например, 7–11 Гц), полосу `ALPHA_BAND` нужно поправить в коде — см.
руководство разработчика.

**Программа не находит фрагментов.**
Смягчите критерии устойчивости (`FREQ_TOL`, `MIN_DURATION`) — см. разработчику.

**Топокарта пустая / просит каналы.**
Для интерполяции по голове нужно минимум 4 отведения с известными координатами.
